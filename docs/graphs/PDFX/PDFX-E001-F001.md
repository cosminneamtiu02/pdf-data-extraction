## Short description

Remove the widget CRUD vertical slice, the database core module, the Alembic directory, the `app/types/` value objects, and the pagination envelope so only the minimal FastAPI shell remains in `apps/backend/app/`.

## Problem

The repository is a monorepo template that currently ships a widget CRUD example on FastAPI + SQLAlchemy + Postgres. The PDF extraction microservice has no database, no persistent storage, and no CRUD surface — every piece of the widget example is dead code the moment the extraction vertical slice starts to land. Leaving it in place would cause three concrete failures for the sole stakeholder (the integrating developer, P-01): `task check` would drag along unrelated failing tests from the removed layers once dependencies shift; `import-linter` contracts would fail on any mixed state where the extraction feature tries to co-exist with a DB-backed slice it cannot coherently import from; and CLAUDE.md's sacred rules forbid patterns the widget example relies on (for example `BaseRepository` and `BaseModel`) so the very act of adding extraction code before the cleanup would put the codebase into a CLAUDE.md-violating state. This feature exists to clear the ground before the new slice arrives, tracing to the Project success criterion **"`task check` passes cleanly on every PR."**

## Scope

- Delete the entire `apps/backend/app/features/widget/` directory (model, repository, service, router, schemas subfolder).
- Delete `apps/backend/app/core/database.py` and `apps/backend/alembic/` plus `apps/backend/alembic.ini`.
- Delete `apps/backend/app/shared/base_repository.py`, `apps/backend/app/shared/base_model.py`, and delete `apps/backend/app/shared/base_service.py` if its interface turns out to be DB-coupled (see open question below).
- Delete `apps/backend/app/types/money.py` and `apps/backend/app/types/currency.py` (widget-era value objects).
- Delete `apps/backend/app/schemas/page.py` (pagination envelope — the extraction service has no paginated endpoints).
- Delete the corresponding widget tests under `apps/backend/tests/` at every level (unit, integration) so `pytest` collects a clean suite after the strip.
- Verify `apps/backend/app/main.py` still boots against the stripped shell and that the `/health` route continues to return 200.

## Out of scope

- Writing any new extraction-feature code (parsers, services, routers, skill loaders). All new code belongs to PDFX-E002 and beyond.
- Modifying `pyproject.toml` or `uv.lock` dependency lists. That's handled by PDFX-E001-F002 in the next feature.
- Touching `infra/compose/docker-compose.yml`, `infra/docker/`, or CI workflow files. Compose is partially addressed by PDFX-E001-F003; CI stays untouched.
- Pruning error codes from `errors.yaml`. That's PDFX-E001-F004.
- Updating `CLAUDE.md`, `docs/architecture.md`, or `Taskfile.yml`. That's PDFX-E001-F005.
- Removing `apps/frontend/` or `packages/api-client/`. That's PDFX-E001-F003.
- Removing Dependabot configuration or the auto-merge workflow. Preserved unchanged.

## Acceptance criteria

- **Given** the repository before the strip, **when** this feature's deletions are applied and `task check` is run, **then** the suite passes with zero errors (no lingering imports, no orphan tests referencing deleted modules, no stale fixtures).
- **Given** the stripped state, **when** `rg "widget" apps/backend/app` is run, **then** zero matches are returned outside of CLAUDE.md (which still references the widget example as a removed-from-template artifact, acceptable).
- **Given** the stripped state, **when** `rg "from sqlalchemy" apps/backend/app` is run, **then** zero matches are returned.
- **Given** the stripped state, **when** `rg "alembic" apps/backend` is run, **then** zero matches are returned.
- **Given** the stripped state, **when** `uvicorn app.main:app` is started against the minimal shell, **then** the process reaches a ready state and `GET /health` returns `200 OK`.
- **Given** the stripped state, **when** `pyright --strict` runs against `apps/backend/app/`, **then** zero errors are reported.
- **Given** the stripped state, **when** `ruff check apps/backend/app/` runs, **then** zero lint errors are reported (including no unused-import warnings from the deletions).

## Dependencies

None within the graph — this feature is the earliest leaf at global priority 10 and has no prerequisites.

## Technical constraints

- **CLAUDE.md sacred rules still apply.** One class per file, no paradigm drift, no `--no-verify`. Deletions are mechanical but must not introduce any CLAUDE.md-forbidden state in the remaining files.
- **Test discipline.** Any tests that exercised the removed modules must be removed, not merely marked skipped. A skipped test is a CLAUDE.md violation ("Never write a test with no assertions" applies to skipped empty shells).
- **No partial deletions.** The widget slice is either fully present or fully gone — leaving one file behind (for example a stale `widget/__init__.py`) will cascade into `pyright` and `import-linter` failures once the new extraction slice lands.
- **Preserve `apps/backend/app/exceptions/base.py`.** The `DomainError` base class is reused by the extraction feature's new errors. Only `_generated/` regenerates in PDFX-E001-F004.
