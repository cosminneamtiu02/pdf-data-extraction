# Benchmark Fixture PDFs

This directory holds the fixture PDF corpus consumed by the local latency
benchmark script (`apps/backend/scripts/benchmark.py`, invoked via
`task bench`). The benchmark sends sequential extraction requests against
a running service — one pass per fixture per output mode — and prints
p50/p95 latency and RSS numbers for manual spot-checking against the
`NFR-004` / `NFR-005` / `NFR-006` / `NFR-008` targets declared in the
requirements spec. Feature traceability: [PDFX-E007-F005](../../../../docs/graphs/PDFX/PDFX-E007-F005.md).

The corpus is intentionally small (~450 KB total across three files) so it
can ship inside the repo without bloating the clone. The fixtures are
**not** used by `task check`; they only exist to serve the opt-in
benchmark loop.

## Provenance

Every file in this directory is **synthetic content authored in-project**
during the PDFX-E007-F005 implementation (landed in commit `67f0e56`, PR
[#71](https://github.com/ioanaecaterinastan-collab/pdf-data-extraction/pull/71)).
None of these PDFs were downloaded, scraped, extracted from a
third-party dataset, or derived from any real business document. All
names, addresses, SKUs, totals, and dates that appear in the rendered
pages are fabricated placeholders (e.g. "Benchmark Supplies Inc.",
"Acme Corp", "INV-2025-0001", "Product 1-1 / SKU-01001"). There is no
personal data, no real trademark usage, and no third-party copyrighted
content inside any of these files.

All three fixtures are therefore covered by the repository's
[MIT License](../../../../LICENSE) (same terms as every other
project-authored artifact in this repo).

The PDFs were generated locally with PyMuPDF (`MuPDF 1.27.2`, visible in
the `/Producer` metadata of each file) as part of building out the
benchmark feature. The generator code was not committed — these files
are treated as opaque test fixtures, regenerated by hand if the shape
of the corpus ever needs to change.

## Regenerating the corpus

There is no committed regeneration script. If the corpus ever needs to
change shape (different page counts, different table layouts, different
invoice fields, etc.), the maintainer should either:

1. Script a regeneration using PyMuPDF or ReportLab under
   `apps/backend/scripts/` (preferred if the corpus becomes a recurring
   churn point), and commit the generator alongside the fixtures so
   provenance stays verifiable from source.
2. Re-author the fixtures ad-hoc and update this README's table to
   match, preserving the synthetic-only / in-project-authored property.

Do **not** replace any of these fixtures with a scraped or downloaded
PDF without first updating this README with a source URL, author
attribution, and a license compatible with MIT redistribution.
